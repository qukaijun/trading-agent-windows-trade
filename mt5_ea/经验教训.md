# 经验教训：MT5 EA 对接实录

## 背景
在 trading-agent-windows-trade 项目中实现 MT5 EA 自动交易信号对接。

## 踩过的坑

### 坑 1：uvicorn + MT5 WebRequest 不兼容
- **现象**：EA 间歇性报 `HTTP status: 1003`
- **原因**：uvicorn 默认启用 HTTP keep-alive，MT5 的 `WebRequest()` 不支持长连接复用
- **解决**：加 `timeout_keep_alive=1` 和 `Connection: close` 响应头
- **教训**：MT5 的 WebRequest 是极简 HTTP 客户端，服务端必须用最保守的 HTTP/1.0 行为

### 坑 2：HTTP 方案不如文件方案可靠
- **现象**：即使修了 keep-alive，MT5 仍可能因网络波动丢失信号
- **解决**：改为 Python 写信号到 `MQL5/Files/ta_signal.txt`，EA 用 `FileOpen` 读取
- **优势**：零网络延迟、永不丢信号、不需要 WebRequest 白名单
- **教训**：本地通信优先文件，不要为了 HTTP 而 HTTP

### 坑 3：Python MT5 API 比 EA 更适合做主控
- **发现**：`MetaTrader5` Python 包可以直接下单、查持仓、读行情
- **优势**：毫秒级响应、不需要编译 EA、调试方便
- **定位**：Python API 做主控（策略→下单），EA 做监控仪表盘

### 坑 4：黄金品种名称因券商而异
- **现象**：XM 券商黄金叫 `GOLD`，不是 `XAUUSD`
- **解决**：`mt5.symbols_get()` 动态获取品种列表
- **教训**：品种代码不能硬编码，必须运行时查询

### 坑 5：MT5 数据目录不在 Program Files
- **现象**：EA 文件拷到 `Program Files\MetaTrader 5\MQL5` 无效
- **原因**：MT5 实际数据在 `%APPDATA%\MetaQuotes\Terminal\<instance>\MQL5\`
- **教训**：MT5 有程序目录和数据目录两个概念，EA/脚本/文件都在数据目录

## 推荐架构（交付给客户）
```
Python 服务端（你的机器/云）
  ├─ 行情分析 → AI 决策
  ├─ 写信号到本地文件 (mt5_bridge.py → MQL5/Files/ta_signal.txt)
  └─ 或直接调 MT5 Python API 下单

客户 MT5
  ├─ FileBridgeEA（读文件自动下单，零配置）
  └─ 或直接让 Python API 控制（需装 Python）
```

## 文件对照
- `mt5_ea/TradingAgentBridgeEA.mq5` — HTTP 轮询版（有 1003 风险，备用）
- `mt5_ea/TradingAgentFileBridgeEA.mq5` — 文件读取版（推荐，可靠）
- `agent_host/mt5_bridge.py` — 服务端信号管理 + 自动写 MT5 Files
