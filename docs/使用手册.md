# 使用手册

## 产品定位

Trading Agent 投研工作台是本地运行的 AI 投研分析工具。当前版本用于个股研究、策略讨论、学习和模拟验证，不开放实盘自动交易。

软件主体基于开源项目集成。我们交付的是本地化部署、模型和数据源配置、培训、运维、诊断和售后支持服务。

## 日常使用流程

1. 双击桌面图标“Trading Agent 投研工作台”。
2. 浏览器打开后，在左侧“模型配置”确认模型供应商、模型名称、Base URL 和 API Key 状态。
3. 输入股票代码或名称。
4. 点击“识别市场”，确认 A 股、港股或美股识别正确。
5. 点击“检查数据源”，确认行情源可用。
6. 点击“生成投研报告”。
7. 在页面右侧阅读中文投研报告。
8. 如需归档，点击“下载 Markdown”“下载 Word”或“下载 PDF”。
9. 后续可在左侧“历史报告”中重新打开最近报告。

## 网页配置

首页左侧“模型配置”支持：

- 选择大模型供应商。
- 填写模型名称。
- 填写 Base URL。
- 填写 API Key。
- 填写 Tushare Token。
- 填写服务方和售后联系方式。
- 保存到本机配置文件。
- 测试模型连接。

API Key 不会在页面明文回显。输入框留空时，系统会保留原 API Key。

## 报告结构

页面优先展示整理后的中文报告，通常包含：

- 报告摘要。
- 标的信息。
- 确定性行情数据。
- 模型结论。
- 核心依据。
- 主要风险。
- 使用边界。
- 原始模型输出。

“确定性行情数据”来自行情数据源，不由大模型自由生成，便于客户区分数据事实与 AI 解读。

美股行情检查依赖第三方公开行情源。若 `PDD` 或 `US:PDD` 在客户电脑上检查失败，通常需要先排查客户网络、第三方数据源限制或临时连接问题，再由服务人员根据诊断包判断是否需要备用数据源方案。

## 历史报告

每次运行分析后，系统会自动把整理后的 Markdown 报告保存到本机：

```text
runtime\reports
```

首页左侧会显示最近的历史报告。点击历史报告后，右侧会重新打开该报告内容。

## 报告导出

当前支持三种导出格式：

- Markdown：适合二次编辑和留存原始结构。
- Word：适合客户交付、人工修订和归档。
- PDF：适合正式发送、展示和只读留存。

下载是附带能力，核心工作流是在页面内阅读、复核和追踪报告。

## 系统诊断

首页左侧“系统诊断”支持：

- 刷新诊断：查看服务状态、配置状态、依赖状态、上游 TradingAgents 状态和历史报告数量。
- 导出诊断：下载文本诊断报告。

如果服务无法启动，客户可从开始菜单点击“导出售后诊断日志”，把生成的压缩包发给服务人员。

## 风险边界

报告只作为投研辅助材料。客户应自行判断，不应把模型输出直接作为买卖依据。
