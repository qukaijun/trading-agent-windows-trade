# 数据源与券商接入路线

## 总原则

首版把“行情/财务/新闻数据”和“实盘交易”分开。行情数据可以先接，实盘交易必须后置。

## Symbol 规范

建议内部统一格式：

```text
A:600519.SH
A:000001.SZ
HK:00700.HK
US:NVDA
US:AAPL
ETF:510300.SH
```

对外输入允许用户写：

```text
600519
600519.SH
00700
0700.HK
腾讯控股
NVDA
英伟达
```

适配层负责归一化，避免港股 `02400` 被误判成 A 股 `002400`。

## 数据源优先级

### A 股

1. Tushare Pro：日线、财务、公告、基础数据
2. Akshare：补充公开行情、板块、指数、特色数据
3. 本地缓存：避免重复调用和限流

### 港股

1. Akshare 港股接口
2. yfinance
3. 后续评估富途 OpenD、长桥 OpenAPI、老虎 OpenAPI

### 美股

1. yfinance
2. Alpha Vantage
3. 后续评估 IBKR、Alpaca、Polygon

## 实盘交易后续候选

### A 股

- QMT
- PTrade
- XTP
- 券商专用 API

A 股实盘接入通常和券商、账户权限、交易终端绑定，不能简单承诺一键通用。

### 港美股

- IBKR
- 富途 OpenD
- 长桥 OpenAPI
- 老虎 OpenAPI
- Alpaca

## 实盘安全要求

如未来开放实盘，至少需要：

- 只读模式和交易模式分离
- 明确授权和二次确认
- 单笔金额限制
- 单日金额限制
- 标的白名单/黑名单
- 下单前预检查
- 操作审计日志
- kill switch
- 异常自动停机
