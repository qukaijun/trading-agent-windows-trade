# Trading Agent Assistant 虚拟机测试流程

适用版本：`0.1.0-mvp`

目标：参考 VibeTrading 的测试方法，在干净 Windows 虚拟机中验证客户拿到安装包后，不需要源码、不需要命令行基础、不需要手动安装 Python/Git/Docker，就能完成安装、启动、配置、生成投研报告、导出诊断。

## 一、测试原则

- 使用干净 Windows 10/11 64-bit 虚拟机。
- 优先测试普通用户体验，再测试命令行细节。
- 安装过程尽量断开外网或弱网观察，确认依赖来自安装包内置资源。
- 不把开发机能跑当成通过标准。
- 每一轮测试都记录安装包文件名、大小、SHA256、Windows 版本、是否预装 Python、错误截图和日志包。

## 四、安装验收

- [ ] 双击安装包。
- [ ] 使用默认安装路径完成安装。
- [ ] 安装过程无红色错误弹窗。
- [ ] 安装后开始菜单存在 `Trading Agent Assistant` 相关入口。
- [ ] 安装后不要求客户手动选择 Python 路径。
- [ ] 安装后不要求客户手动安装 Git、Docker、Node 或 Visual Studio 编译工具。
- [ ] 安装目录存在 `scripts\install-offline.ps1`、`scripts\start.ps1`、`scripts\repair.ps1`、`scripts\export-logs.ps1`。
- [ ] 安装目录存在 `agent_host\server.py`、`agent_host\ui.py`、`wheelhouse\`、`vendor\upstream\TradingAgents\`。

## 五、入口冒烟测试

在安装目录打开 PowerShell，运行：

```powershell
.\scripts\verify-entrypoints.ps1
```

通过标准：

- [ ] `pip check` 通过。
- [ ] Python 依赖导入通过。
- [ ] `/health` 返回正常。
- [ ] 首页 `/` 返回 HTTP 200。
- [ ] `/docs` 返回 HTTP 200。
- [ ] A股、港股、美股样例代码识别通过。
- [ ] `/api/diagnostics` 返回正常。
- [ ] `stop.ps1` 可以停止服务。

## 六、界面验收

- [ ] 双击开始菜单或桌面快捷方式能打开浏览器。
- [ ] 首页地址为 `http://127.0.0.1:8787/`。
- [ ] 首屏突出“研究任务”和“投研报告”，不是配置后台。
- [ ] 报告区域无内部滚动条，使用整页滚动。
- [ ] 历史报告打开后显示报告目录。
- [ ] 报告按章节分区显示。
- [ ] 原始模型输出或错误信息等附录默认折叠。
- [ ] 页面没有明显中文乱码。
- [ ] 页面没有横向滚动条或文字重叠。

## 七、模型配置验收

- [ ] 打开“模型与服务配置”。
- [ ] 选择模型供应商，例如通义千问 / DashScope。
- [ ] 填写模型名、Base URL、API Key。
- [ ] 保存配置成功。
- [ ] API Key 留空保存时不会清空原配置。
- [ ] 测试模型连接能给出成功或明确失败原因。
- [ ] 页面和诊断不明文显示完整 API Key。

## 八、投研业务验收

建议最小测试集：

```text
A股：600519、000001、300750
港股：00700.HK、9988.HK
美股：AAPL、NVDA
```

逐项验证：

- [ ] 点击“识别市场”返回正确市场和标准代码。
- [ ] 点击“检查数据源”返回可读结果。
- [ ] 点击“生成投研报告”后有明确运行状态。
- [ ] 成功时生成中文分区投研报告。
- [ ] 失败时生成可读错误报告，不直接把 Python Traceback 当客户主界面。
- [ ] 报告进入历史记录。
- [ ] 点击历史记录能重新打开报告。
- [ ] Markdown 导出正常。
- [ ] Word 导出正常。
- [ ] PDF 导出正常。

## 九、诊断与售后验收

- [ ] 首页“服务支持”可刷新诊断。
- [ ] `/api/diagnostics.txt` 可打开。
- [ ] 开始菜单“Export Support Logs”可导出日志包。
- [ ] 日志包不包含完整 API Key。
- [ ] `Repair Installation` 可运行。
- [ ] `Stop Service` 可运行。

## 十、失败场景

- [ ] 未配置 API Key 时，报告生成失败提示清楚。
- [ ] 填错 API Key 时，测试模型返回可读失败原因。
- [ ] 网络不可用时，数据源或模型错误提示清楚。
- [ ] 输入不存在的股票代码时，市场识别或数据源检查有提示。
- [ ] 上游 TradingAgents 不可用时，报告仍能生成“分析未完成”的错误报告。
