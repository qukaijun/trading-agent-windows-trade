# Trading Agent 底座选型报告

## 结论

建议首版以 TauricResearch/TradingAgents 作为核心研究引擎参考，以 TradingAgents-CN 作为中文产品化和 A/HK/US 市场适配参考。不要直接复制中文增强版的完整 Web 架构进入 Windows MVP，否则 MongoDB、Redis、Docker 和前端构建链会增加普通用户安装成本。

推荐方案：

- 核心 agent：参考原版 TradingAgents
- 中文体验：参考 TradingAgents-CN
- Windows 交付：自研轻量外壳
- 市场接入：自研统一数据适配层
- 实盘交易：后续单独版本，不进 MVP

## 原版 TradingAgents

优点：

- 上游活跃，版本持续更新
- 多智能体结构清晰：基本面、情绪、新闻、技术分析、研究辩论、交易员、风控、组合经理
- 支持多模型供应商和本地模型
- 适合作为核心研究引擎
- Apache-2.0 许可证更适合商业封装

注意点：

- 默认体验更偏英文和开发者
- 数据源对中国市场不是完整产品化体验
- 一键安装需要额外封装
- 需要确认每个版本的 Python、依赖和 Windows 兼容性

## TradingAgents-CN

优点：

- 中文本地化成熟度更高
- 已经面向 A 股、港股、美股做了很多适配
- 有 Web UI、配置管理和学习中心思路
- 适合参考中文用户体验和市场规则处理

注意点：

- 架构较重：FastAPI + Vue + MongoDB + Redis + Docker
- Windows 单机一键安装会更复杂
- 需要仔细核对许可证、第三方依赖和数据源条款
- 社区 issue 中可见港股/美股数据源稳定性问题，需要做降级和容错

## MVP 推荐取舍

首版不建议做完整 Web 全栈。推荐先做：

- 本地 API 服务
- 自动打开浏览器的简洁中文页面
- 中文配置向导
- 统一数据源测试页
- 报告导出
- 日志导出

等核心稳定后，再考虑 Vue 前端、缓存服务、数据库服务和 Docker 镜像。
