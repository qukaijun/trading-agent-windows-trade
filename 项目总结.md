# 项目总结与改进路线图

## 一、做了什么

### 1. 调研参考竞品
登录分析了 **孙越AI (xauusd.team)**：
- PHP + Bootstrap 网页端，用户注册登录体系
- 核心卖点：上传 MT5 截图 → GPT-5 / Gemini / Claude 多模型并行看图分析
- MT5 三审 EA 框架，HTTP 轮询信号
- 免费额度 + VIP 付费模式

### 2. 利用现有资产
工作区已有 `trading-agent-windows-mvp`：
- TradingAgents 多智能体投研框架
- yfinance / Tushare / FMP 多数据源
- FastAPI + 中文 Web UI
- MT5 信号桥（仅模拟盘）
- 一键安装打包
- MCP 工具集

### 3. 新建交易版项目
创建 **`trading-agent-windows-trade`** 独立仓库：

| 改动 | 文件 |
|------|------|
| 黄金 XAUUSD 支持 | `markets.py` → 映射到 GOLD/GC=F |
| COMMODITY 市场类型 | `markets.py` + `capabilities.py` + `data_source_registry.py` |
| MT5 实盘开关 | `server.py`：`MT5_LIVE_TRADING_ENABLED` 环境变量 |
| 实盘风控 | 止损必填、量上限 `MT5_MAX_LIVE_VOLUME` |
| 文件版 EA | `TradingAgentFileBridgeEA.mq5` — 读本地文件，不调 HTTP |
| 自动写信号文件 | `mt5_bridge.py` → `MQL5/Files/ta_signal.txt` |

### 4. MT5 实盘验证
- Python `MetaTrader5` 包直连 MT5，当场下一单 EURUSD + 一单 GOLD
- EA HTTP 轮询踩坑：uvicorn keep-alive 导致 MT5 报 1003
- 最终方案：**Python API 直连做主控，比 EA HTTP 轮询更快更稳**

### 5. GitHub 版本管理
- 原版 `qukaijun/trading-agent-windows-mvp` — 投研分析，不碰实盘
- 交易版 `qukaijun/trading-agent-windows-trade` — 贵金属+加密货币+实盘
- 经验教训记录在 `mt5_ea/经验教训.md`

## 二、架构定位

```
TradingAgents 多智能体（AI 大脑）
  ├─ 市场分析师 + 技术分析师
  ├─ 风险分析师 + 研究经理
  └─ 辩论 → 决策（买/卖/等）
         │
         ▼
Python mt5_bridge（手）
  └─ MetaTrader5 API 直接下单
         │
         ▼
客户 MT5 终端
  ├─ 模拟盘：零风险验证
  └─ 实盘：需开 MT5_LIVE_TRADING_ENABLED
```
